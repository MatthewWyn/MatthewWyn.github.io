I started from complete scratch with web dev in Novemer, 2017. These are some things I learnt by finally building the FreeCodeCamp tribute site.

After countless lessons and tutorials on FreeCodeCamp, CodeCademy, Udemy, YouTube, SitePoint, Medium, Lynda... I finally got round to doing the first FreeCodeCamp project of building a tribute site! But in these short few months I've also discovered some incredible teachers (I've narrowed it down to my favourite two) on Udemy and YouTube. Although there are many good resources online I still felt like I needed a structure ("To know what I needed to know") and the teacher I put my trust into was Colt Steele. And more recently Brad Schiff. These guys are incredible teachers, there is something they just GET about approaching every concept from the eyes of a beginner. They use repetition of learning material in a really creative way (by building several mini-projects that reuse the same things you've been learning. They talk slowly, they don't waffle and they keep you engaged).

But how do you know if you're getting anywhere?
Sean Smith (posting on Medium) has some great advice on making you realise that you should hone in on what works for you. (I know that phrase is thrown around like an old clich√©!).

"To improve your learning process you have to become very cognizant of what works for you, how you learn effectively, and how to locate the best resources for your learning style. Then, you can begin to optimize your learning approach based on these observations. For example, I knew that I benefited from watching video tutorials and coding alongside them."


The things I learnt making the tribute site (In other words: The things that still hadn't sunk in by only watching tutorials and code-alongs):
  • How to use the "background-size: cover" so that the header image is cropped and cut to fit, but if it has plenty of room then it shows more of the image. This gives a nice responsive and "zoomy" feel to the site.
  • How to use the CSS grid (On YouTube I watched Net Ninja's excellent and free CSS Grid course)
  • I discovered that a CSS framework link rel has to come before your stylesheet in the HTML header
  • I learnt how to fix a problem where I couldn't display the hover pop-up text on the gallery images without cutting off the text... box-sizing: border-box; solved it! (This was from looking at W3Schools examples, which I recommend playing with. I even used it to double check the Bootstrap jumbotron code for this page!)
  • I learnt that CSS Grid is preffered by many web developers over Flexbox even though CSS Grid is relatively new. But also, any developers say that CSS Grid is good for some things and Flexbox is good for other things. I am learning them both.

Some final thoughts

I think learning is a "mess". You learn 5 things here, 1 thing there. Then it seems like you learn nothing at all for 3 days straight. Then you suddenly surprise yourself. I don't think this will really change as I continue studying. However, courses are still very useful because they give you structure and show you the path of things that you need to be working on, even if you end up disliking how one teacher teaches Flexbox (you can always find another tutorial).
